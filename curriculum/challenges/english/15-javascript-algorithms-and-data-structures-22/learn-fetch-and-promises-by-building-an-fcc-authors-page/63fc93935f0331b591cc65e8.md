# --description--

We will now get into using the `fetch` API. It is a built-in JavaScript interface for getting a resource from a server. It has a `fetch()` method with which you can make a `GET`, `POST`, `PUT`, or `PATCH` request. The request we are making in this project is a `GET` request to this endpoint `https://cdn.freecodecamp.org/curriculum/news-author-page/authors.json`. The endpoint has the information of freeCodeCamp News Site authors – making it ideal for what we want to do.

To make a `GET` request with the `fetch()` method, the first thing you need to do is to put the endpoint inside the curly braces of the `fetch()` method. The request you make with `fetch()` is always asynchronous – meaning other parts of your code can keep running when the request is still running.

The `fetch()` method returns a `Promise`, which is an important aspect of asynchronous JavaScript. The `Promise` is either `fulfilled` or `rejected`. You can use the `.then()` method to access the `Promise` if it fulfils and the `.catch()` method to handle errors when it rejects.

Make a `GET` request to the freeCodeCamp authors endpoint. Don't terminate the code with a semi-colon yet.
