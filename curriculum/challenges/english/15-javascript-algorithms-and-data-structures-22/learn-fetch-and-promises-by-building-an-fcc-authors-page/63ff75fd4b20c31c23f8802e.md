# --description--

Now that you have the data you want, what's next is to populate it on the UI. But the endpoint returns an array of 26 authors, you won't want to populate all of them on the UI at the same time for better performance.

What you should do instead is to populate 8 authors at a time. And then add a functionality to populate 8 more on each demand until there's no data to display anymore.

Create 2 variables named `startingIndex` and `endingIndex`, and assign them the number values `0` and `8` respectively. Also, create a `authorDataArr` variable and set it to an empty array.

**Note**: You will use the `startingIndex` and `endingIndex` variables to paginate the authors, so you can get only 8 at first. You will also put the data (an array) into the empty array of the `authorData` soon.
